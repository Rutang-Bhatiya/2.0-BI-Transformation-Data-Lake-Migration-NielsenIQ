## 2.1 Legacy Reporting (Answers) vs Discover (BI Platform)

Legacy tools were highly structured and report-centric.  
Discover introduced a **data model‚Äìdriven, exploratory analytics approach**.

### Key Differences

| Aspect | Legacy (Answers) | Discover |
|-----|-----------------|----------|
| Data Access | Predefined extracts | Centralized data lake |
| Report Style | Fixed layouts | Flexible, exploratory |
| Logic | Embedded in reports | Model-driven |
| Scalability | Limited | Enterprise-scale |

The migration required **rethinking reports**, not copy-pasting them.

---

## 2.2 Report Redesign & Business Logic Translation

Reports were rebuilt to:
- Answer the **same business questions**
- Avoid replicating outdated layouts
- Leverage Discover‚Äôs strengths

### Redesign Principles
- Focus on **business intent**, not visual similarity
- Simplify KPIs where possible
- Use standardized metrics from the data model
- Reduce dependency on manual logic

This often meant telling customers:
> ‚ÄúYou will get the same answer, but not the same old report.‚Äù

---

## 2.3 Data Governance, Masking & Access Rules

![Governance & Masking]

Discover introduced stricter **data governance**.

### Responsibilities

- Clarified why certain data was:

This required balancing:
- Transparency
- Security
